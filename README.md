## Seccion de Markdown

---

1. Titulos y subtitulos 
    - Sirve para estructurar y jerarquizar el contenido ded un documento, mejorando su legibilidad y organizacion

2. Texto en negritas y cursivas
    - Su uso consiste en resaltar palabras clave o conceptos muy importante, captanto la atencion del lector, mientras que la cursiva se usa para dar enfasis

3. Listas ordenadas y no ordenadas
    - Es para enumerar elementos donde el orden es importante, como pasos de un proceso, instrucciones o cualquier secuencia logica

4. Enlaces
    - Para insertar hipervinculos en texto plano de forma sencilla y legible permitiendo conectar documentos entre si, ademas de que no altere el orden y limpieza de la documentacion

5. Imagenes
    - Sirve para insertar contenido visual (Como graficos), permitiendo que el documento sea legible de una manera mas sencilla y facil de compartir la informacion

8. bloques de codigo
    - Se usa principalmente para mostrar fragmentos de codigo fuente, conservando el formato original (evitnado saltos, errors, espacios) y evitando que el markdown interprete caracteres especiales

## Seccion de Comandos Git

---

1. git status 
    - sirve principalmente para ver el status dee un repositorio de git asi como tambien muestra los archivos modificados
2. git add
    - sirve principalmente para agregar archivos al area de preparacion antes de ser enviados
3. git add "<nombre del archivo>"
    - permite anadir, editar o eliminar un archivo individual dependiendo del comando git en cuestion, en este caso "add" anade un archivo
4. git add -A
    - permite seleccionar todos los archivos
5. git commit -m
    -m sirve para agregar un mensaje al ultimo commit enviado, aunque tambien podemos usar --amend para modificar el mensaje guardado
6. git push
    - sirve para enviar los datos que se prepararon a github
7. git branch ->nombre< -d -checkout -switch
    - ` ` `git branch >nombre de la rama<` ` ` sirve especificamente para crear la rama
    - ` ` `git checkout o git switch >nombre de la rama<` ` ` para cambiar entre las ramas
    - ` ` `git branch -d >nombre de la rama<` ` ` para eliminar la rama en cuestion
8. git revert/reset
    -  ` ` `git revert` ` ` deshace los cambios del ultimo commit
    -  ` ` `git reset` ` ` borra los commits del historial local, puede ser riesgoso si ya se compartio anteriormente
